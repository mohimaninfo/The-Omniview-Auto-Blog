How to Build a Local AI Assistant with Ollama and Open-Source LLMs

Alex Chen AI Correspondent, TechDepth June 12, 2025 11 min read

Overview

Running a large language model (LLM) on your own hardware has gone from a research-lab privilege to a weekend project achievable on a consumer laptop. Tools like Ollama — an open-source runtime launched in 2023 — have reduced the complexity of local model deployment to a few terminal commands.[1] As of mid-2025, Ollama supports over 90 model families including Meta's Llama 3, Mistral, Google's Gemma 2, and Microsoft's Phi-3.[2]

This guide walks you through the complete process: installing Ollama, choosing the right model for your hardware, running your first conversation, exposing the local REST API, and connecting a browser-based UI. Every step is free — no cloud API keys, no subscriptions, and your data never leaves your machine.

Privacy note: Unlike cloud AI services, local models process all prompts on your own CPU or GPU. No conversation data is transmitted to external servers. This makes local LLMs ideal for working with sensitive documents, code, or personal notes.

Prerequisites

✅ Before You Begin

  • RAM: 8 GB minimum (16 GB recommended for 7B+ parameter models)
  • Storage: 5–15 GB free disk space per model
  • OS: macOS 12+, Linux (Ubuntu 20.04+), or Windows 10/11
  • GPU (optional): NVIDIA (CUDA), AMD (ROCm), or Apple Silicon — dramatically improves speed
  • Terminal: Basic command-line familiarity

Step-by-Step Instructions

1

Install Ollama

Ollama provides a single installer for all platforms. It bundles the model runtime, a CLI, and a local API server that starts automatically.[1]

macOS / Linux:

curl -fsSL https://ollama.ai/install.sh | sh

Windows: Download the installer from ollama.ai and run the .exe file.

After installation, verify it's running: 

ollama --version
# Expected: ollama version 0.x.x
2

Choose and Pull a Model

Model size determines quality vs. speed. For most laptops, a 7B or 8B parameter model is the sweet spot. Meta's Llama 3 8B achieved state-of-the-art performance among open-weight models in its class when released in April 2024.[3] 

# Pull Llama 3 8B (4.7 GB download)
ollama pull llama3

# Lighter option for older hardware (2.0 GB)
ollama pull phi3:mini

# Coding-optimized model
ollama pull codellama

Browse all available models at ollama.ai/library.

Local model inference keeps data on-device, unlike cloud inference which routes data through external servers. Credit: Wikimedia Commons (CC0)

3

Run Your First Chat

Start an interactive conversation directly in your terminal:

ollama run llama3

>>> Hello! Can you explain how transformers work in simple terms?
# The model responds instantly from local inference

Type /bye to exit the session. The model stays loaded in memory for ~5 minutes after your last message, making subsequent conversations faster.

4

Use the Local REST API

Ollama runs a local HTTP server at http://localhost:11434, exposing endpoints compatible with OpenAI's API format.[4] You can query it from any programming language or tool: 

curl http://localhost:11434/api/chat -d '{
  "model": "llama3",
  "messages": [
    { "role": "user", "content": "Write a Python function to reverse a string." }
  ],
  "stream": false
}'

This means you can build your own AI-powered apps, scripts, or automations that run entirely offline. Any library supporting OpenAI's SDK works with Ollama by pointing the base URL to localhost:11434.

5

Connect a Browser UI (Open WebUI)

For a full ChatGPT-like experience in your browser, install Open WebUI — a self-hosted interface with conversation history, model switching, document upload, and multi-modal support.[5] 

# Requires Docker Desktop (free)
docker run -d -p 3000:8080 \
  --add-host=host.docker.internal:host-gateway \
  -v open-webui:/app/backend/data \
  --name open-webui \
  ghcr.io/open-webui/open-webui:main

Open http://localhost:3000 in your browser. On first launch, create an admin account, select your pulled model, and start chatting.

Performance Tips & Optimizations

Hardware Acceleration

On Apple Silicon Macs, Ollama automatically uses the Neural Engine via Metal GPU acceleration — no configuration needed. On Linux with an NVIDIA GPU, install CUDA 12.1+ and Ollama will use it automatically. Expect 3–10× faster token generation compared to CPU-only inference.[1]

Memory Management

Ollama loads models into RAM/VRAM on first use and keeps them resident for 5 minutes of idle time. To keep a model loaded persistently: 

# Set OLLAMA_KEEP_ALIVE to -1 to keep model in memory indefinitely
OLLAMA_KEEP_ALIVE=-1 ollama run llama3

💡 Pro Tips

  • Use ollama list to see all downloaded models and their disk size.
  • Use ollama rm model_name to delete models and free disk space.
  • Try mistral:7b for strong general performance at the 7B size.
  • For coding tasks, deepseek-coder:6.7b outperforms base models.
  • Set OLLAMA_HOST=0.0.0.0 to make the API accessible on your local network.

Conclusion

Local AI assistants powered by Ollama and open-source LLMs represent a fundamental shift in how individuals and teams can access AI capabilities — without cloud dependencies, API costs, or privacy concerns. The ecosystem has matured rapidly: as of 2025, open-weight models from Meta, Mistral AI, Google, and Microsoft routinely match or exceed proprietary models from 2023 across standard benchmarks.[3]

Whether you're building a private document Q&A system, a local coding assistant, or simply exploring AI without sending your data to the cloud, Ollama provides the most accessible on-ramp available today.

Frequently Asked Questions

Do I need a GPU to run local LLMs with Ollama?
No — Ollama runs on CPU-only machines. However, a GPU dramatically improves token generation speed. On a modern M2 MacBook Pro, Llama 3 8B generates around 40–60 tokens per second. On CPU-only hardware, expect 5–15 tokens per second depending on your processor.
Is Ollama free to use commercially?
Ollama itself is open-source (MIT license) and free for commercial use. Individual models have their own licenses — Llama 3, for example, is free for commercial use up to 700 million monthly active users under Meta's community license. Always verify the specific license of each model you use.
How does Ollama compare to running models via Python (llama.cpp)?
Ollama uses llama.cpp under the hood, but wraps it in a much more user-friendly package: automatic model management, a REST API server, and cross-platform installers. For most users, Ollama is the better choice. Advanced users who need fine-grained control over quantization or inference parameters may prefer direct llama.cpp usage.
Can I use Ollama with Python or JavaScript code?
Yes. The official ollama Python library and ollama-js npm package provide idiomatic wrappers. You can also use the OpenAI SDK by setting base_url="http://localhost:11434/v1" and any placeholder API key — Ollama ignores authentication on localhost.
What's the best model for coding assistance?
As of mid-2025, deepseek-coder-v2 and codellama:13b are leading choices for code generation. For lighter hardware, phi3:mini offers impressive coding capability at just 2.3 GB. Pull any of these with ollama pull model_name.
